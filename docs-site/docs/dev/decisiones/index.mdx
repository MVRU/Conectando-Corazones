Cada decisión técnica relevante se registra como un **Architecture Decision Record** (ADR).

## Numeración

- Usá números secuenciales de tres dígitos (`01`, `02`, ...).
- El nombre del archivo sigue el formato `NNN.mdx`.
- El front matter incluye `title: "NNN - Título"`.

## Estructura

1. **Contexto**: describe el problema o necesidad.
2. **Alternativas**: lista de opciones evaluadas.
3. **Decisión**: explicación de la solución elegida.
4. **Consecuencias**: impactos positivos y negativos.
